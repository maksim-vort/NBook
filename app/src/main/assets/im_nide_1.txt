В этом уроке мы рассмотрим систему библиотек NIDE 2017.
Библиотеки в NIDE - фрагменты кода, регулярно использующиеся в реальных проектах. Для удобства в папке с библиотекой находится сам файл с кодом (lib.js) и файл с основными данными о библиотеке (info.nlib).
Файл с кодом библиотеки содержит объект, в котором находится все функции и переменные библиотеки. ВНИМАНИЕ! ВСЕ переменные должны быть ВНУТРИ объекта, для избегания конфликтов!
Файл со свойствами библиотеки содержит следующие строчки:
<~code>nide-api:1
library-version:1.0
keyword:Name1
keyword:Name2</code>
где Name1 и Name2 - названия функций; nide-api - уровень АПИ NIDE, пока что всегда 1, позже он будет увеличиваться; libary-version - версия библиотеки. Практически не нужна)))
Существует два типа библиотек - встроенные библиотеки NIDE и пользовательские библиотеки. Встроенные библиотеки изменять нельзя - иначе ваш проект нельзя будет перенести на другое устройство или открыть после переустановки программы. Пользовательские библиотеки - часть проекта и их можно создавать, изменять и даже предлагать мне (создателю NIDE) сделать их встроенными.
Если данная библиотека для нормальной работы требует другую библиотеку, ее следует подключить в описании библиотеки следующей строчкой
<~code>requires-library:Location/Name</code>
где Location - nide (для встроенных библиотек) или project (для библиотек проекта).

Для того, чтобы подключить встроенную библиотеку к моду, можно воспользоваться пунктом меню "Project/Manage libraries", где можно выбирать, какие библиотеки нужны в проекте. 
Пользовательские библиотеки, созданные с помощью пункта меню "Project/New library", автоматически подключаются в проект.
Так же можно напрямую изменить набор библиотек вручную, в файле проекта (.nproj), просто удалить или дописать нужные строчки формата
<~code>include-library:Location/Name</code>
